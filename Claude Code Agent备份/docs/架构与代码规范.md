## 架构与代码规范

- **项目结构**: 严格遵循标准Maven/Gradle项目布局。采用清晰的分层架构：
  - `controller/` - REST API入口，仅负责HTTP协议处理与DTO转换
  - `service/` - 业务逻辑层，纯Java代码，无框架依赖
  - `mapper/` - 数据访问层，面向接口编程
  - `domain/` - 领域模型（Entity, VO, Domain Events），核心业务逻辑所在
- **错误处理**: **[强制]** 所有异常必须保留完整异常链（Exception Chaining）。禁止直接`e.printStackTrace()`或`log.error(e.getMessage())`后吞掉异常。业务异常必须继承`RuntimeException`并传递`cause`。API错误响应应遵循RFC 7807 (ProblemDetail)规范。使用Spring时，用`@ControllerAdvice`和`@ExceptionHandler`统一处理异常。
- **日志**: **[强制]** 使用SLF4J作为日志门面，Logback或Log4j2作为实现。必须采用**结构化日志**（JSON格式）输出。关键上下文信息（如`traceId`, `userId`, `requestId`）必须通过MDC（Mapped Diagnostic Context）注入到每条日志中。
- **接口设计**: 遵循"面向接口编程"与"依赖倒置"原则。优先定义小的、单一职责的接口（Interface Segregation）。鼓励使用Java Records作为DTO，使用Sealed Classes处理受限继承。
- **现代Java特性**: 优先使用`java.time`（JSR-310）处理日期时间，使用`Optional`处理可空值，使用Stream API进行集合操作，使用Pattern Matching（Java 17+）简化类型判断，使用Virtual Threads（Java 21+）处理高并发I/O，使用MapStruct进行对象转换。
- **设计模式**: 适当使用设计模式，避免过渡设计，以“代码是给人看的”为原则，遵循单一职责、开闭、里氏替换、接口隔离和依赖倒置原则。
- **注释**: 重要的字段和复杂的方法需要有注释，方法注释上包含`@param`,`@return`,`@throws`和描述文字，新增的类注释包含`@author`和描述。
